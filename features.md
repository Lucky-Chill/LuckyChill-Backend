## 1. 프로젝트 개요

---

### 한 줄 정의

통합 강의실 예약 시스템

### 제품 목표

강의실 사용 현황 조회 및 예약 과정을 간소화하여 학생, 교수, 조교의 편의성을 극대화하고, 강의실 자원의 효율적인 운영을 통해 학내 자원 관리의 투명성과 생산성을 높이는 것을 목표로 합니다.

### 배경

현재 강의실 사용 현황과 예약 정보가 분리되어 수동으로 관리되고 있어, 조교는 시간표와 예약 신청을 개별적으로 확인하는 데 많은 시간과 노력이 소요됩니다. 학생과 교수 또한 강의실의 실제 사용 가능 시간을 파악하고 예약하는 과정에서 번거로움을 겪고 있으며, 이는 자원 활용의 비효율성과 사용자 불편을 초래하고 있습니다.

### 사용자 문제

사용자(학생, 교수)는 강의실별 실제 수업 시간표와 예약 가능 시간을 한눈에 확인하기 어렵고, 빈 강의실을 찾아 예약하는 과정이 번거롭습니다. 관리자(조교)는 강의실 시간표 확인과 예약 신청을 수동으로 처리해야 하는 업무 부담이 큽니다.

### 해결 방안

강의실별 수업 시간표와 예약 현황을 통합하여 보여주는 웹 기반 예약 시스템을 제공합니다. 사용자는 직관적으로 빈 시간을 확인하고 예약 신청할 수 있으며, 관리자는 신청 내역을 효율적으로 승인 또는 반려할 수 있도록 하여 전체 과정을 간소화하고 자동화합니다.

### 차별점

기존의 분산되고 수동적인 관리 방식에서 벗어나, 학교 계정 기반의 통합 웹 시스템을 통해 강의실 사용 현황과 예약 기능을 한곳에 모아 제공합니다. 이를 통해 학생, 교수, 조교 모두에게 통일된 사용자 경험을 제공하고, 투명하고 효율적인 자원 관리를 실현하여 타 시스템 대비 월등한 편의성과 업무 효율성을 제공합니다.

### 타겟 사용자

주요 타겟 사용자는 학내 강의실을 사용하고자 하는 '학생'입니다. 특히 스터디, 팀 프로젝트, 동아리 활동 등을 위해 강의실 대여가 필요한 학생들이 편리하게 이용할 수 있도록 합니다. 또한, 수업 외의 목적으로 강의실 사용이 필요한 '교수' 및 '조교' 또한 관리자 역할 및 일반 사용자 역할을 수행합니다.

### 사용자 시나리오

학생이 팀 프로젝트를 위해 빈 강의실을 찾아야 할 때, 시스템에 접속하여 특정 강의실의 주간 시간표를 확인합니다. 현재 진행 중인 수업이 없는 빈 시간을 확인한 후, 원하는 시간대를 선택하여 예약 신청을 합니다. 이후 '내 예약 신청 상태 확인' 기능을 통해 자신의 예약이 승인되었는지 확인합니다.

### 핵심 지표

월간 강의실 예약 완료 건수 (최소 20% 증가), 시스템을 통한 예약 비율 (전체 예약 중 70% 이상), 사용자 만족도 점수 (4점 이상/5점 만점), 조교의 강의실 예약 관리 업무 시간 감소 (20% 이상).

### 리스크

낮은 사용자 채택률 (기존 수동 방식 고수), 시스템 오류로 인한 중복 예약 발생, 학교 시스템과의 연동 문제, 데이터 보안 및 개인정보 유출 위험, 관리자의 신규 시스템 적응 저항.

### 카테고리

> 생산성 / 업무

### 사용자 역할

> 관리자, 사용자

### 디바이스 (기기)

> Web

## 1. 학교 계정 기반 사용자 인증

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자는 학교 Google 계정을 통해 시스템에 로그인할 수 있어야 합니다. 로그인 시, 계정 정보(이메일 등)를 기반으로 조교 계정인 경우 관리자 페이지로, 그 외 계정은 사용자 페이지로 자동 진입합니다.

**수용 기준:**

```
1. ○ 사용자는 학교 Google 계정으로 성공적으로 로그인할 수 있어야 합니다.
2. ○ 등록된 조교 계정으로 로그인 시, 관리자 페이지로 자동 리다이렉트되어야 합니다.
3. ○ 일반 사용자 계정으로 로그인 시, 사용자 페이지로 자동 리다이렉트되어야 합니다.
```

### 1.1 Google OAuth 연동 로그인 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자, 관리자 | **기기**: Web

사용자가 학교 Google 계정을 사용하여 시스템에 안전하게 로그인 할 수 있도록 Google OAuth 인증 방식을 지원합니다.

### 1.2 조교 계정 자동 관리자 페이지 연결

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 관리자 | **기기**: Web

로그인 시 조교로 등록된 계정은 자동으로 관리자 전용 페이지로 리다이렉트되어 관리자 기능에 접근할 수 있습니다.

### 1.3 일반 사용자 자동 사용자 페이지 연결

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Web

일반 사용자(학생, 교수) 계정으로 로그인 시, 전용 사용자 페이지로 자동 연결되어 일반 예약 기능을 이용할 수 있도록 합니다.

## 2. 강의실 시간표 및 예약 현황 조회

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자는 특정 강의실의 주간 시간표와 실시간 예약 현황을 한눈에 확인할 수 있어야 합니다. 수업 시간과 예약된 시간을 명확히 구분하여 표시합니다.

**수용 기준:**

```
1. ○ 사용자는 강의실 목록에서 특정 강의실을 선택하여 주간 시간표를 조회할 수 있어야 합니다.
2. ○ 시간표는 수업 시간과 이미 예약된 시간을 시각적으로 구분하여 표시해야 합니다.
3. ○ 각 시간 블록에 대한 상세 정보(예: 수업명, 예약자 등)를 확인할 수 있어야 합니다.
```

### 2.1 강의실 목록 및 시간표 조회 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Web

사용자는 웹 화면에서 강의실 목록을 확인하고, 원하는 강의실을 선택하여 해당 강의실의 주간 수업 시간표와 예약 현황을 동시에 조회할 수 있습니다. 시간표는 수업 중인 시간과 예약된 시간이 색상이나 아이콘 등으로 명확히 구분되어 시각적으로 쉽게 인식할 수 있도록 합니다.

### 2.2 시간표 내 상세 정보 확인 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Web

사용자는 시간표의 각 시간 블록을 클릭하거나 터치하여 해당 시간에 진행 중인 수업명, 예약자 명, 예약 목적 등 자세한 정보를 팝업 또는 별도 창으로 확인할 수 있습니다.

## 3. 강의실 예약 신청

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자는 조회된 시간표에서 빈 시간대를 선택하여 강의실 예약 신청을 할 수 있어야 합니다. 예약 신청 시, 사용 목적 등의 정보를 함께 제출합니다.

**수용 기준:**

```
1. ○ 사용자는 빈 시간 블록을 클릭하여 예약 신청 양식을 불러올 수 있어야 합니다.
2. ○ 예약 신청 시 사용자 목적, 사용 인원, 연락처 등 필수 정보를 입력할 수 있어야 합니다.
3. ○ 예약 신청이 성공적으로 제출되면 사용자에게 확인 메시지가 표시되어야 합니다.
```

### 3.1 빈 시간대 예약 신청 인터페이스

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Web

사용자는 주간 시간표에서 빈 시간 블록을 클릭하여 예약 신청 양식을 간편하게 불러올 수 있어야 합니다. 예약 창은 직관적인 UI로 예약 가능 시간을 명확히 표시합니다.

### 3.2 예약 신청 정보 입력 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Web

사용자는 예약 신청 시 사용 목적, 사용 인원, 연락처 등 필수 정보를 입력할 수 있는 양식을 제공받아야 합니다. 입력값의 유효성을 실시간으로 검증하여 오류를 최소화합니다.

### 3.3 예약 신청 제출 및 확인 알림

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Web

사용자가 예약 신청을 완료하면 시스템은 즉시 성공 메시지를 표시하고, 신청 내역을 사용자 마이페이지에 자동 등록하여 신청 상태를 확인할 수 있도록 합니다.

## 4. 내 예약 신청 상태 확인

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자는 자신이 신청한 강의실 예약 내역과 현재 승인/반려 상태를 확인할 수 있어야 합니다.

**수용 기준:**

```
1. ○ 사용자는 마이페이지 또는 별도 메뉴에서 자신의 예약 신청 목록을 확인할 수 있어야 합니다.
2. ○ 각 예약 건에 대해 '승인 대기', '승인 완료', '반려' 상태가 명확히 표시되어야 합니다.
3. ○ 반려된 예약의 경우 반려 사유를 확인할 수 있어야 합니다.
```

### 4.1 내 예약 신청 목록 조회 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Web

사용자는 마이페이지 또는 별도 메뉴에서 자신이 신청한 모든 강의실 예약 내역을 목록 형태로 조회할 수 있어야 합니다. 목록에는 예약 강의실명, 예약 날짜 및 시간, 신청 일자가 포함되어야 합니다.

### 4.2 예약 승인 상태 표시 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Web

사용자는 각 예약 내역에 대해 '승인 대기', '승인 완료', '반려' 등 현재 상태가 명확하게 표시되는 UI를 통해 예약 상태를 쉽게 구분할 수 있어야 합니다.

## 5. 관리자 예약 신청 승인/반려

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

관리자는 제출된 모든 강의실 예약 신청 목록을 조회하고, 각 신청에 대해 승인 또는 반려 처리할 수 있어야 합니다. 반려 시 반려 사유를 입력할 수 있습니다.

**수용 기준:**

```
1. ○ 관리자는 모든 예약 신청 목록을 최신순으로 조회할 수 있어야 합니다.
2. ○ 각 예약 신청 건에 대해 '승인' 또는 '반려' 버튼이 제공되어야 합니다.
3. ○ 예약 반려 시 관리자는 필수적으로 반려 사유를 입력할 수 있어야 합니다.
4. ○ 승인/반려 처리 후 사용자에게 상태 변경에 대한 알림이 전송되어야 합니다.
```

### 5.1 예약 신청 목록 신청일자 순 조회 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 관리자 | **기기**: Web

관리자는 제출된 모든 강의실 예약 신청 목록을 신청일자 순서대로(오래된 신청 건부터 먼저 조회) 정렬하여 확인할 수 있어야 합니다. 이를 통해 승인 처리 전 예약들은 먼저 신청한 순서대로 검토할 수 있도록 하여 공정성과 업무 효율성을 높입니다.

### 5.2 예약 승인 및 반려 처리 인터페이스

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 관리자 | **기기**: Web

관리자는 각 예약 신청 건에서 '승인'과 '반려' 버튼을 통해 간편하게 상태를 변경할 수 있습니다. 반려 시에는 필수 입력란으로 반려 사유를 작성하도록 하여 명확한 커뮤니케이션이 이루어지도록 지원합니다.

## 6. 관리자 강의실 및 수업 시간표 관리

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

관리자는 강의실 목록을 등록, 수정, 삭제할 수 있으며, 각 강의실의 실제 수업 시간표를 등록 및 수정할 수 있어야 합니다.

**수용 기준:**

```
1. ○ 관리자는 새로운 강의실 정보를 시스템에 등록할 수 있어야 합니다.
2. ○ 기존 강의실의 정보(이름, 수용 인원 등)를 수정하거나 삭제할 수 있어야 합니다.
3. ○ 각 강의실별로 정규 수업 시간표를 등록, 수정, 삭제할 수 있어야 합니다.
4. ○ 등록된 수업 시간표는 사용자 조회 화면에 즉시 반영되어야 합니다.
```

### 6.1 수업 시간표 등록 및 관리 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 관리자 | **기기**: Web

관리자는 각 강의실별로 정규 수업 시간표를 등록, 수정, 삭제할 수 있어야 하며, 생성 및 변경 사항은 즉시 사용자 조회 화면에 반영되어야 합니다.

### 6.2 강의실별 시간표 및 예약 승인 실시간 자동 동기화 기능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 관리자, 사용자 | **기기**: Web

관리자가 강의실 시간표를 수정하는 것뿐만 아니라, 사용자가 신청한 예약이 관리자의 승인을 받으면 해당 예약 내역이 실시간으로 강의실 시간표에 즉시 반영되어야 합니다. 이를 통해 시간표와 승인된 예약 정보가 항상 최신 상태로 유지되어, 사용자들이 최신 강의실 사용 현황을 실시간으로 확인할 수 있도록 지원합니다.
